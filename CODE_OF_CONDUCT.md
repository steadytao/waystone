# Code of Conduct

## Purpose

Waystone is intended to be a serious project-history portability tool.

The purpose of this Code of Conduct is to keep project discussion, review and collaboration constructive, professional and usable for everyone involved.

## Expectations

Participants in the Waystone project are expected to:
- act in good faith
- communicate clearly and professionally
- criticise ideas, code and decisions without attacking people
- respect project scope, standards and review processes
- avoid disruptive, abusive or needlessly hostile behaviour
- accept that maintainers may make final decisions about project direction

## Unacceptable Behaviour

The following behaviour is not acceptable in project spaces:
- harassment or intimidation
- personal attacks
- discriminatory or degrading language
- bad-faith disruption
- repeated refusal to follow documented project processes
- intentionally misleading technical discussion
- publication of private information without consent

## Scope

This Code of Conduct applies to:
- issues
- pull requests
- discussions
- review comments
- other repository-associated project spaces

## Maintainer Responsibility

Maintainers are responsible for enforcing this Code of Conduct reasonably and proportionately.

They may edit, hide, lock, remove content or restrict participation where necessary to protect the project and its contributors.

## Reporting

Serious conduct concerns should be reported privately to `mail@steadytao.com`.

## Notes

This Code of Conduct is intended to support a functional working environment, not performative process.

The project values direct technical discussion but not abusive or corrosive behaviour.
