Waystone v0.2.0-beta.1 proves the migration contract across multiple forge namespaces and documents the identity rules that keep provenance intact. This is the first beta checkpoint in the v0.2 line.

## Highlights

- Multi-forge conformance coverage now exercises GitHub, GitLab, Forgejo, Gitea and local `waystone:` records in one ledger.
- The conformance flow runs `migrate report`, `migrate plan`, `migrate inspect` and `migrate verify` against the same multi-source history.
- `docs/migration-identity.md` documents the rule that original source identity is evidence and target identity is a projection.

## Security

- No known security fixes in this release.

## In This Release

- GitHub, GitLab, Forgejo and Gitea source records are validated together without merging source-local numbers.
- Local `waystone:` continuation records are validated alongside imported forge records.
- Migration plans are checked for source-scoped target keys across all supported imported forge namespaces.
- Migration identity documentation explains source IDs, canonical Waystone IDs, target keys, author ambiguity, label ambiguity and milestone ambiguity.
- README, architecture and roadmap documentation now link the migration identity contract.

## Release Line

`v0.2.0-beta.1` is not the stable v0.2 contract. The next v0.2 checkpoint should add a read-only strategy file with safe defaults and a structured migration loss report.
