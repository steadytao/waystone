This alpha release establishes Waystone's first usable local ledger surface for preserving GitHub project history. It is still experimental but it can import, audit, browse, verify, sign and export repository history without yet becoming a hosted forge or CI system.

## Highlights

- GitHub import and exit-readiness audit commands for preserving collaboration records and reporting migration blockers.
- Deterministic `.waystone/` ledger storage with source manifests, operation records, strict verification and archive import/export.
- Local Ed25519 signing for operation records, source manifests and archive manifests with an explicit local trust policy.

## Security

- No known security fixes in this release.

## In This Release

- Added GitHub OAuth device-flow authentication and `GITHUB_TOKEN` override support.
- Added GitHub repository import for issues, pull requests, comments, labels, milestones, releases and review comments.
- Added GitHub exit-readiness audit evidence for workflows, templates, policy files, repository settings and migration-relevant GitHub surfaces.
- Added local browsing for issues, pull requests, comments, timelines, labels, milestones, sources, audit records and operation history.
- Added ledger verification, doctor, status, diff, archive inspection, archive export and archive import commands.
- Added release infrastructure for platform archives, checksums, SBOMs, Sigstore bundles and provenance attestations.
- Added the initial documentation set covering CLI usage, ledger format, operations, signing, privacy, security, architecture decisions and self-import validation.
