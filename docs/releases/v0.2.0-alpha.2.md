Waystone v0.2.0-alpha.2 makes saved migration plans multi-source. The plan artefact now preserves source namespaces directly, carries collision and ambiguity warnings and keeps target projections source-scoped instead of implying that matching source-local numbers are the same record.

## Highlights

- `waystone migrate plan` now accepts repeated `--from` values, matching the cross-source shape already used by `migrate report`.
- Migration plan records now include explicit source namespaces and source-scoped target keys.
- Saved plans now carry the same collision and ambiguity warnings used by migration reports.

## Security

- No known security fixes in this release.

## In This Release

- Single-source migration planning remains supported.
- Multi-source migration planning is still read-only and does not contact or mutate target forges.
- `preserve-source-numbering` remains the only supported numbering strategy.
- Plan records sort deterministically across source namespaces so generated artefacts are stable.
- README, CLI, roadmap, changelog and ADR documentation now describe multi-source migration planning.

## Release Line

`v0.2.0-alpha.2` is not the stable v0.2 contract. The next v0.2 checkpoint is plan inspection and plan verification, so saved plans can be reviewed and validated before any bridge or export dry-run work exists.
