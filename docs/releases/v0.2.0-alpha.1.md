Waystone v0.2.0-alpha.1 starts the v0.2 bridge-ready migration-contract line. It captures the current portable collaboration ledger shape: imported forge history from GitHub, GitLab, Forgejo and Gitea, local issue continuation under `waystone:` sources, strict verification, archive export/import, migration reports, saved read-only migration plans and cross-source migration reports.

## Highlights

- Read-only GitLab, Forgejo and Gitea import paths with separate source namespaces and token handling.
- Migration reports and saved migration plans, including cross-source reports with collision and ambiguity warnings.
- Improved CLI help and usage output with concrete arguments and options instead of generic placeholders.

## Security

- No known security fixes in this release.

## In This Release

- Local labelled issue round-trip validation now covers strict verification, archive export, archive inspection, archive import and imported-ledger verification.
- `waystone migrate report` reports preserved records, local continuation records, known gaps and cross-source ambiguity without mutating a target forge.
- `waystone migrate plan` writes a deterministic read-only migration plan using `preserve-source-numbering`.
- GitHub API errors preserve token-scope and documentation details where GitHub returns them.
- GitLab import uses `gitlab:` source provenance and keeps GitLab merge request identity distinct from GitHub pull request terminology.
- Forgejo and Gitea imports are separate commands with separate source provenance and token environment variables.
- `waystone help <command>` and `waystone <command> help <subcommand>` now provide command-specific help across the practical CLI surface.
- User-facing usage output now uses concrete required and optional arguments instead of generic `[flags]`, `[options]` or `[args]`.

## Release Line

`v0.2.0-alpha.1` is not the stable v0.2 contract. It is the first checkpoint in the v0.2 line. The v0.2 line should finish multi-source migration plans, plan inspection, plan verification, structured loss reporting, conformance fixtures and compatibility policy before `v0.2.0`.
