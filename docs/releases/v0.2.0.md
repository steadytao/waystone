Waystone v0.2.0 is the first bridge-ready migration-contract release. It keeps Waystone local and read-only with respect to target forges, while making the ledger, archive and migration-plan surfaces explicit enough for a separate bridge tool to consume.

## Highlights

- Finalises the v0.2 compatibility surface for ledger archives, source namespaces, migration plans, strategy files, loss reports and conformance fixtures.
- Preserves project history from GitHub, GitLab, Forgejo, Gitea and local `waystone:` sources without merging source-local identity or numbering.
- Hardens archive, ledger, source-inspection and credential-store file handling around symlink and path-safety boundaries.

## Security and Correctness

- Trusted operation and source-manifest signatures require the signature public key to match the stored trusted identity public key.
- Ledger-owned extracted, copied and private-key files use exclusive creation.
- Ledger reads reject symlinks, non-regular files and replacement races.
- Source inspection uses rooted no-symlink opens for manifest and object hashes.
- Plaintext credential-store operations reject symlinked roots and token paths.
- `golang.org/x/sys` is updated to a version that clears `GO-2026-5024`.

## In This Release

- `waystone version` reports `0.2.0`.
- `docs/compatibility.md` defines the implemented contract for `waystone.ledger.v1`, `waystone.archive.v1`, `waystone.migration_plan.v1`, `waystone.migration_strategy.v1` and `waystone.migration_loss_report.v1`.
- `testdata/conformance/` covers single-source, multi-source, local Waystone, migration-plan, strategy-file and loss-report fixtures.
- Migration plan verification rejects unsupported source systems, unsupported object types, duplicate JSON members, trailing JSON, invalid source namespaces, source/target mismatches, negative source numbers and unsafe target writes.
- Issue comments and pull request / merge request conversation comments are separated using parent-object metadata.
- Archive inspection and import enforce defensive entry-count, per-entry and aggregate-size limits.
- Safe import remotely confirms `github:` sources only in v0.2; GitLab, Forgejo, Gitea and local Waystone sources rely on archive integrity, source manifests and strict verification.

## Scope

`v0.2.0` is still local-first project-history tooling. It does not mutate target forges, sync changes, host attachments, execute imported data, run CI, provide federation or act as a production security boundary.
