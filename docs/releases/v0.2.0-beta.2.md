Waystone v0.2.0-beta.2 makes migration policy explicit and adds structured loss reporting. This is the final beta feature checkpoint before the v0.2 release-candidate work.

## Highlights

- `waystone migrate plan --strategy-file <file>` now accepts a versioned strategy file with only safe read-only defaults.
- `waystone migrate loss-report --json` now emits machine-readable migration loss categories.
- Migration documentation now describes strategy files and structured loss reports as part of the v0.2 bridge-ready contract.

## Security

- No known security fixes in this release.

## In This Release

- Strategy files use `waystone.migration_strategy.v1` and reject unsupported versions, unknown fields and unsafe strategy values.
- Strategy files currently accept only the same safe defaults as generated migration plans, including `target_write_strategy = none`.
- Structured loss reports use `waystone.migration_loss_report.v1`.
- Loss reports cover attachments, review-thread semantics, CI history, workflows, permissions, branch protections, user mapping, release assets and visibility uncertainty.
- `migrate loss-report` is read-only. It does not contact a target forge and does not write operation records.

## Release Line

`v0.2.0-beta.2` is not the stable v0.2 contract. The next v0.2 checkpoint should add conformance fixtures and a compatibility policy before `v0.2.0-rc.1`.
