Waystone v0.2.0-alpha.3 makes migration plans reviewable and independently verifiable. Saved plans are now artefacts that Waystone can inspect for humans and validate before any future bridge or export dry-run consumes them.

## Highlights

- `waystone migrate inspect <plan>` prints a human-readable summary of a saved migration plan.
- `waystone migrate verify <plan>` validates the plan version, safe strategies, required fields, source namespaces and deterministic target keys.
- Plans generated by `waystone migrate plan` now have a direct validation path before any target-specific tooling exists.

## Security

- No known security fixes in this release.

## In This Release

- `migrate inspect` rejects unknown plan versions unless `--allow-unknown` is supplied.
- `migrate verify` rejects duplicate records with the same source, object and source ID.
- `migrate verify` rejects target write strategies other than `none`.
- `migrate verify` rejects target keys that do not match `preserve-source-numbering`.
- README, CLI, roadmap, changelog and ADR documentation now describe plan inspection and verification.

## Release Line

`v0.2.0-alpha.3` is not the stable v0.2 contract. The next v0.2 checkpoint should add a multi-forge conformance fixture and migration identity documentation.
